# File Operations in Bash

## The Filing Cabinet Analogy

An office manager needs to work with physical files every day: check if a folder exists before filing a document, open a file and read its contents line by line, add new entries to a log book, and archive old records. All of these actions have direct equivalents in Bash.

Working with files is one of the most common tasks in shell scripting. Log parsing, configuration management, CSV processing, backup scripts — all of them require reading from and writing to files.

```
  File operations in Bash:

  Check existence    [[ -f "file" ]]
        |
        v
  Read line by line  while read -r line
        |
        v
  Write/append       echo "text" > file
        |             echo "text" >> file
        v
  Process content    grep, awk, sed, cut
        |
        v
  Delete/move        rm, mv, cp
```

## Checking Files and Directories

Always check before operating:

```bash
#!/usr/bin/env bash

CONFIG="/etc/myapp/app.conf"
LOG_DIR="/var/log/myapp"

# Check if file exists
if [[ ! -f "$CONFIG" ]]; then
    echo "ERROR: Config file not found: $CONFIG"
    exit 1
fi

# Check if directory exists, create if missing
if [[ ! -d "$LOG_DIR" ]]; then
    echo "Creating log directory: $LOG_DIR"
    mkdir -p "$LOG_DIR"
fi

# Check if file is readable
if [[ ! -r "$CONFIG" ]]; then
    echo "ERROR: Cannot read: $CONFIG"
    exit 1
fi

# Check if file is non-empty
if [[ ! -s "$CONFIG" ]]; then
    echo "WARNING: Config file is empty: $CONFIG"
fi
```

## Writing to Files

```bash
# Overwrite (create or truncate)
echo "Starting deployment at $(date)" > /var/log/deploy.log

# Append (add to existing)
echo "Step 1: pulling image" >> /var/log/deploy.log
echo "Step 2: stopping old container" >> /var/log/deploy.log

# Write multiple lines with here document
cat > /etc/myapp/config.conf << 'EOF'
# Generated by deploy.sh
APP_PORT=8080
DB_HOST=localhost
LOG_LEVEL=INFO
MAX_CONNECTIONS=100
EOF

# Write variable content to file
backup_manifest="/tmp/backup_manifest.txt"
{
    echo "# Backup Manifest - $(date)"
    echo "hostname=$(hostname)"
    echo "version=$APP_VERSION"
    echo "files=$(find /app -name '*.py' | wc -l)"
} > "$backup_manifest"
```

## Reading Files Line by Line

The canonical Bash pattern for reading a file line by line:

```bash
while IFS= read -r line; do
    echo "Line: $line"
done < /etc/hosts
```

Breaking down the components:

```
  while IFS= read -r line; do
        ^^^  ^^^^^^^
        |        |
        |        +-- read -r: raw mode (no backslash processing)
        |            line: variable that holds each line
        |
        +-- IFS=: set Internal Field Separator to empty string
                  prevents leading/trailing whitespace from being stripped
```

Why `IFS=`? Without it, lines like `"  indented content"` would have their leading spaces stripped.

```bash
# Practical: read a config file, skip comments and blank lines
while IFS= read -r line; do
    # Skip blank lines
    [[ -z "$line" ]] && continue
    # Skip comment lines
    [[ "$line" =~ ^[[:space:]]*# ]] && continue
    echo "Config line: $line"
done < /etc/myapp/app.conf
```

### Reading file into array

```bash
# Method 1: one element per line
mapfile -t lines < /etc/hosts
echo "Total lines: ${#lines[@]}"
echo "First line: ${lines[0]}"

# Method 2: process substitution
mapfile -t users < <(grep "home" /etc/passwd | cut -d: -f1)
```

## Parsing Key=Value Configuration Files

```bash
#!/usr/bin/env bash
# load_config.sh — parse KEY=VALUE format config

load_config() {
    local config_file="$1"
    declare -gA CONFIG

    if [[ ! -f "$config_file" ]]; then
        echo "ERROR: Config file not found: $config_file" >&2
        return 1
    fi

    while IFS='=' read -r key value; do
        # Skip blank lines and comments
        [[ -z "$key" || "$key" =~ ^[[:space:]]*# ]] && continue

        # Trim whitespace
        key="${key#"${key%%[![:space:]]*}"}"
        key="${key%"${key##*[![:space:]]}"}"
        value="${value#"${value%%[![:space:]]*}"}"
        value="${value%"${value##*[![:space:]]}"}"

        # Remove surrounding quotes from value
        value="${value#\"}"
        value="${value%\"}"
        value="${value#\'}"
        value="${value%\'}"

        CONFIG["$key"]="$value"
    done < "$config_file"
}

# Example config file (app.conf):
# DB_HOST=localhost
# DB_PORT=5432
# APP_NAME="My Application"

load_config "app.conf"
echo "DB_HOST: ${CONFIG[DB_HOST]}"
echo "DB_PORT: ${CONFIG[DB_PORT]}"
```

## Parsing CSV Files

```bash
#!/usr/bin/env bash
# process_servers.csv format:
# hostname,ip_address,role,environment
# web01,10.0.1.10,webserver,production
# db01,10.0.2.20,database,production

CSV_FILE="servers.csv"
HEADER_SKIPPED=false

while IFS=',' read -r hostname ip_address role environment; do
    # Skip the header row
    if [[ "$HEADER_SKIPPED" == false ]]; then
        HEADER_SKIPPED=true
        continue
    fi

    # Skip blank lines
    [[ -z "$hostname" ]] && continue

    echo "Processing: $hostname ($ip_address) — $role in $environment"

    # Do something with each server
    if [[ "$environment" == "production" ]]; then
        echo "  -> Adding to production monitoring"
    fi
done < "$CSV_FILE"
```

## Temporary Files

Always clean up temporary files:

```bash
#!/usr/bin/env bash

# Create a safe temporary file
TEMP_FILE=$(mktemp /tmp/deploy-XXXXXX.txt)
TEMP_DIR=$(mktemp -d /tmp/build-XXXXXX)

# Always clean up, even on error (covered in traps section)
trap 'rm -f "$TEMP_FILE"; rm -rf "$TEMP_DIR"' EXIT

# Use the temp file
echo "Build started: $(date)" > "$TEMP_FILE"
cp -r /app/src "$TEMP_DIR/"

# Process and use...
cat "$TEMP_FILE"
ls "$TEMP_DIR"

# Cleanup happens automatically via trap
```

## Finding and Processing Files

```bash
#!/usr/bin/env bash
# log_cleanup.sh — archive and rotate log files

LOG_DIR="/var/log/myapp"
ARCHIVE_DIR="/var/log/archive"
MAX_AGE_DAYS=30

mkdir -p "$ARCHIVE_DIR"

# Find and process files older than 30 days
while IFS= read -r -d '' logfile; do
    filename=$(basename "$logfile")
    echo "Archiving: $filename"
    gzip -c "$logfile" > "$ARCHIVE_DIR/${filename}.gz"
    rm "$logfile"
done < <(find "$LOG_DIR" -name "*.log" -mtime "+$MAX_AGE_DAYS" -print0)

# Count remaining logs
remaining=$(find "$LOG_DIR" -name "*.log" | wc -l)
echo "Log rotation complete. $remaining log files remain."
```

The `-print0` and `read -d ''` pattern handles filenames with spaces correctly.

## Real-World Example: Log Analyzer

```bash
#!/usr/bin/env bash
# analyze_logs.sh — parse nginx access log, report on errors

LOG_FILE="${1:-/var/log/nginx/access.log}"
REPORT_FILE="/tmp/log_report_$(date +%Y%m%d).txt"

if [[ ! -f "$LOG_FILE" ]]; then
    echo "ERROR: Log file not found: $LOG_FILE"
    exit 1
fi

echo "Analyzing: $LOG_FILE"
echo ""

# Count total lines
total_lines=$(wc -l < "$LOG_FILE")

# Count error responses
error_4xx=$(grep -c '" 4[0-9][0-9] ' "$LOG_FILE" 2>/dev/null || echo 0)
error_5xx=$(grep -c '" 5[0-9][0-9] ' "$LOG_FILE" 2>/dev/null || echo 0)

# Find top 5 IPs
echo "Top 5 client IPs:"
awk '{print $1}' "$LOG_FILE" | sort | uniq -c | sort -rn | head -5

# Write report
{
    echo "Log Analysis Report - $(date)"
    echo "File: $LOG_FILE"
    echo "Total requests: $total_lines"
    echo "4xx errors:     $error_4xx"
    echo "5xx errors:     $error_5xx"
} > "$REPORT_FILE"

echo ""
echo "Report saved to: $REPORT_FILE"
```

## Summary

> **Always check** if files exist with `-f`, `-d`, `-e` before operating on them.
>
> **`while IFS= read -r line; do ... done < file`** is the canonical pattern for reading files line by line.
>
> **`IFS=`** prevents whitespace stripping. **`-r`** prevents backslash processing. Both are important.
>
> **`>`** creates or overwrites. **`>>`** appends. **`cat > file << 'EOF'`** writes multi-line blocks.
>
> **`mapfile -t array < file`** reads an entire file into an array (one element per line).
>
> **`mktemp`** creates safe temporary files. Always pair with a `trap EXIT` to clean up.

| Operation | Syntax |
|-----------|--------|
| Check file exists | `[[ -f "$file" ]]` |
| Check dir exists | `[[ -d "$dir" ]]` |
| Write (overwrite) | `echo "text" > file` |
| Write (append) | `echo "text" >> file` |
| Read line by line | `while IFS= read -r line; do` |
| Read into array | `mapfile -t arr < file` |
| Create temp file | `tmp=$(mktemp)` |
| Create temp dir | `tmpdir=$(mktemp -d)` |
